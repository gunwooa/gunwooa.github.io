[Gatsby] [Gatsby] Gatsby 프로젝트에서 TypeScript를 사용하는 방법

2022.01.09 · 2 min read

💡 들어가며

Gatsby 프로젝트에서 TypeScript를 사용하는 방법입니다. 해당 포스트에서는 Gatsby CLI와 gatsby new를 통해 새로운 프로젝트를 시작했습니다. TypeScript를 사용할 때 필요한 Gatsby Plugin 설치 및 설정 방법과 절대경로 설정 내용도 포함하고 있습니다.


1. 플러그인 설치하기

Gatsby 프로젝트에서 TypeScript를 사용하기 위해서는 TypeScript와 Gatsby 플러그인 설치가 필요합니다.


  • npm
npm i --save-dev typescript npm i gatsby-plugin-typescript
  • yarn
yarn add typescript --dev yarn add gatsby-plugin-typescript

2. Gatsby config 파일 수정하기

설치 이후에는 gatsby-config.js 파일에 TypeScript 설정 내용을 작성해야 합니다. gatsby-config.js 파일에 다음과 같이 작성해주세요. 상세 옵션에 대한 설명은 @babel/preset-typescript 문서에서 확인하실 수 있습니다.

module.exports = { ... plugins: [ ... { resolve: 'gatsby-plugin-typescript', options: { isTSX: true, allExtensions: true, }, }, ... ], };

3. tsconfig 파일 작성하기

TypeScript를 설정하는 데 필요한 tsconfig.json 파일을 생성하겠습니다.


  • npm
tsc --init
  • yarn
yarn tsc --init

tsconfig.json 파일을 생성한 후 다음과 같이 작성했습니다.

{ "compilerOptions": { "target": "es5", "module": "commonjs", "allowJs": true, "jsx": "preserve", "strict": true, "noUnusedLocals": true, "noUnusedParameters": true, "noImplicitReturns": true, "baseUrl": "./src", "paths": { "assets/*": ["./assets/*"], "components/*": ["./components/*"], "styles/*": ["./styles/*"], "templates/*": ["./templates/*"], "typings/*": ["./typings/*"] }, "esModuleInterop": true, "skipLibCheck": true, "forceConsistentCasingInFileNames": true, "resolveJsonModule": true }, "include": ["src/**/*.ts", "src/**/*.tsx"], "exclude": ["node_modules"] }

위에서 작성한 거처럼 baseUrl을 ./src로 설정하고 path 항목에 사용하는 폴더 이름을 작성하면 src 폴더 내의 파일을 절대경로로 사용할 수 있습니다.


절대경로를 사용하면 src > components > Layout > AppLayout 파일의 Layout 컴포넌트를 호출할 때

import Layout from 'components/Layout/AppLayout'

다음과 같이 작성하면 src 폴더 내의 모든 파일에서 위치에 상관없이 Layout 컴포넌트를 사용할 수 있습니다. 상대경로를 사용하는 것 보다 상대적으로 깔끔한 코드를 작성할 수 있습니다.


또한, 위와 같이 설정한 baseUrl을 사용하기 위해 추가로 webpack 설정이 필요합니다.


gatsby-node.js 파일에 다음과 같이 작성해주세요.

const path = require('path') // Setup Import Alias exports.onCreateWebpackConfig = ({ getConfig, actions }) => { const output = getConfig().output || {} actions.setWebpackConfig({ output, resolve: { alias: { assets: path.resolve(__dirname, 'src/assets'), components: path.resolve(__dirname, 'src/components'), styles: path.resolve(__dirname, 'src/styles'), templates: path.resolve(__dirname, 'src/templates'), typings: path.resolve(__dirname, 'src/typings'), }, }, }) }

다음과 같이 tsconfig.json, gatsby-node.js 파일을 설정하면 절대경로로 파일을 사용할 수 있습니다.


👏 마치며

Gatsby 프로젝트에서 TypeScript를 사용하는 방법에 대해 알아봤습니다. 프로젝트에서 TypeScript를 사용한다면 절대경로 설정을 추천합니다. 사용하지 않을 때 보다 훨씬 편하게 코드를 작성할 수 있습니다. 👍
